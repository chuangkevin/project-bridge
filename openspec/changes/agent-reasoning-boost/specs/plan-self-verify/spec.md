## ADDED Requirements

### Requirement: JSON plan is verified by a self-check AI call
After the JSON plan is generated by `callAIJSON`, the system SHALL make one additional Flash API call to verify the plan. The verification prompt checks for: orphan pages (not linked by any other page), missing navigation paths, pages with empty/vague descriptions, and pages that don't match the discussed page list.

#### Scenario: Orphan page detected
- **WHEN** plan has page "設定頁" but no other page has "設定頁" in its navigationOut
- **THEN** self-check flags it and adds navigation links from relevant pages

#### Scenario: Page missing from plan
- **WHEN** Echo's confirmation said 5 pages but JSON plan only has 4
- **THEN** self-check adds the missing page with a reasonable spec

#### Scenario: Plan passes verification
- **WHEN** all pages have navigation, no orphans, descriptions are adequate
- **THEN** self-check returns the plan unchanged

### Requirement: Self-check returns corrected JSON plan
The verification call SHALL use JSON mode and return a corrected plan in the same format as the original. The corrected plan replaces the original before generation proceeds.

#### Scenario: Correction applied
- **WHEN** self-check adds a missing page
- **THEN** the corrected plan (with the extra page) is used for generation, not the original

### Requirement: Self-check failure is non-blocking
If the self-check API call fails (429, timeout, parse error), the system SHALL log a warning and proceed with the original unchecked plan.

#### Scenario: API failure
- **WHEN** self-check call returns 429
- **THEN** system logs "[plan-verify] Failed, using original plan" and continues

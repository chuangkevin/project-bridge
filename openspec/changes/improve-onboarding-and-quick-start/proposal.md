## Why

目前使用者第一次進入 `project-bridge` 時，容易不知道應該先做什麼。

雖然系統已具備：

- 顧問模式
- 設計模式
- 架構模式
- 文件上傳
- MCP / skills / 元件庫

但空狀態主要仍是「沒有資料」式提示，而不是「下一步該怎麼開始」的任務導向引導。

這造成幾個實際問題：

- 使用者不知道哪個模式適合當前任務
- 使用者不知道第一句 prompt 應該怎麼寫
- 有文件的人不知道應該先做審查還是先生成畫面
- 空白工作區沒有清楚的第一個成功路徑

## What Changes

- 在首頁與工作區加入任務導向 quick-start 引導
- 提供可直接點擊的起手任務卡與範例 prompt
- 讓使用者能用最少認知成本開始：
  - 分析需求
  - 生成第一版畫面
  - 先聊流程再做設計

## Non-Goals

- 不在這個 change 內完成完整 onboarding wizard
- 不在這個 change 內改寫所有模式命名與 IA
- 不在這個 change 內處理長期教學內容或說明中心

## Success Criteria

- 新使用者在首頁或空工作區能立即看到 2-3 個明確起手方式
- 至少一個 quick-start 入口可直接將 prompt 注入對話並開始執行
- 顧問 / 設計 / 架構的主要用途在空狀態中更清楚

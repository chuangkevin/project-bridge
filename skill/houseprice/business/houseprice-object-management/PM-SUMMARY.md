# 物件管理 — PM 業務摘要

## 這是什麼

007 平台的核心功能 — 不動產經紀人在此刊登物件（不動產），付費後消費者可以在 Buy 平台看到。

## 使用者流程

```
經紀人登入 007 → 購買額度 → 填寫物件資訊 → 扣額度刊登 → 消費者在 Buy 平台看到
```

- 刊登是**付費**的，額度像代幣，一個額度 = 一個物件一段刊登期
- 到期後物件從 C 端**完全消失**（不是標記已下架）
- 經紀人可加購廣告服務提升曝光

## 物件的生命週期

```
草稿 → 庫存（已上傳）→ 公開（C端可見）→ 到期/關閉/強制下架
                      ↗ 非公開（經紀人隱藏）
```

## 物件關閉的原因

| 原因 | 觸發方式 |
|------|---------|
| 刊登到期 | 自動（廣告期限到期） |
| 自行關閉 | 經紀人手動 |
| 客服強制下架 | 客服操作（**不可恢復**） |
| 會員刪除 | 會員帳號刪除時自動關閉所有物件 |
| 變更仲介店 | 系統處理 |
| 仲介店未通過審核 | 系統處理 |

## 功能變更影響評估

新增或修改物件欄位時，影響大小取決於**欄位要出現在哪裡**：

| 影響等級 | 欄位用途 | 需要協調的範圍 |
|---------|---------|--------------|
| 🟢 小 | 只在 B 端庫存頁面顯示 | 只改一個專案 |
| 🟡 中 | 要在 C 端搜尋結果顯示 | 改 2-3 個專案 + 搜尋引擎 |
| 🔴 大 | 要參與歸戶判斷 | 需跟 DBA 協調 |

## 廣告加值服務

| 服務 | 說明 | 對搜尋排序影響 |
|------|------|--------------|
| 關注焦點 | 焦點位廣告 | 最高 |
| 優質推薦 | 推薦位廣告 | 次高 |
| AI影音 | AI 生成影片 | 有 |
| AI美裝 | AI 照片美化 | 有 |
| 定時更新 | 自動刷新排序 | 無（刷新機制，不加權重） |

## 其他

- 物件數量沒有上限
- 租屋有獨立的管理系統
- 經紀人可追蹤競品物件（Track 功能）
- 消費者留言後系統通知經紀人
- 定價邏輯需由 PM 提供
